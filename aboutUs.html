ICS Project 2 GitHub Tutorials

Welcome to the tutorial page for ICS Project 2. This guide helps students collaborate with Git and GitHub, manage branches, submit pull requests, and keep team code clean, trackable, and review-friendly.

Beginner Friendly Team Workflow Project Submission Ready

Tutorial Goals

  • Create and clone a repository for Project 2.
  • Use branches for feature-based development.
  • Write meaningful commit messages.
  • Open pull requests and perform peer review.

Starter Commands

Run these commands in your terminal after creating your repository on GitHub.

git clone https://github.com/your-org/ics-project-2.git cd ics-project-2 git checkout -b feat/your-feature-name

Daily Workflow

  1. Pull latest changes from the main branch.
  2. Implement your task in a feature branch.
  3. Commit in small, logical chunks.
  4. Push and open a pull request for review.

Commit Message Style

Use a simple convention to make history easy to read:

feat: add login form validation fix: resolve merge conflict in navbar docs: update setup instructions

Common GitHub Tasks in Project 2

  • Assign teammates as reviewers on pull requests.
  • Use pull request descriptions to explain your changes.
  • Resolve merge conflicts before requesting a final review.
  • Tag release milestones if your instructor requires checkpoints.
